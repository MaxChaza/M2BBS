Siphon/Trap Computation

Here it is possible to choose between computing siphons or computing traps by clicking on the corresponding radio button.

Siphon

A siphon is a set of places with the property that the set of post-transitions is a subset of (or equal to) the set of pre-transitions. If the set of post-transitions is a real subset of the set of pre-transitions, then the siphon is called bad siphon.

Trap

A trap is a set of places with the property that the set of pre-transitions is a subset of (or equal to) the set of post-transitions.

Options

By selecting the checkbox "export siphons" (resp. "export traps") one can choose a file to that the siphons (traps) shall be exported. If the checkbox "proper sets" is selected then only the proper sets are computed and exported (if exporting is selecing).

When computing siphons is selected then one can check the loaded Petri net for the siphon-trap-property (also known as "deadlock-trap-property") by selecting the checkbox "stp". A Petri net has the sipho-trap-property if every siphon includes an initially marked trap. By definition it is obvious that an empty Petri net, never fullfills the siphon-trap-property.

Place Set Analyzer

By clicking on the button "place set analyzer" a new window opens in that the name of each place is listed. One can click on the names of the places to manually check if the selected places are a siphon or a trap. It is also shown if the set of places selected is a bad siphon. There are three options with that either the names of the places are shown, the ids of the places or both is shown.
