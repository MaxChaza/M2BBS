IM Based Analysis

In this dialog one can setup analyzers that are based on the incidence matrix of the loaded Petri net. The different analyzers are

  • rank theorem: to start the analysis if the rank theorem holds

  • check structural boundedness: to start the analysis if the Petri net is structurally bounded

  • p-invariants: to start the analysis of p-invariants; there are several options that can be setup

  • t-invariants: to start the analysis of t-invariants; there are several options that can be setup

Rank Theorem

The rank theorem is a theorem from that one can make conslusions about the liveness behavior of a Petri net. The property in this context is that the rank-theorem-property holds if the rank of the incidence matrix is equal to the cardinal number of set of structuraly conflicts, i.e., if rank(IM) = |SCCS| - 1 holds.

The rules that can be applied can be looked up in the rule set help.

Check Strurual Boundedness

If a Petri net has the structural boundedness property it is bounded with any marking. The property is validated by solving a linear programming problem using the simplex algorithm.

The rules that can be applied can be looked up in the rule set help.

P-Invariants

Laxly speaking a P-invariant of a Petri net is a set of places where the sum of the tokens on the places is always equal. Formally, a P-invariant is a vector p so that p C = 0, where C is the incidence matrix.

There are several options for the calculation of invariants.

  • delete trivial invariants: removes all trivial invariants from the list, i.e., only those invariants are listed that are not trivial
  • check coverage: an invariant covers the Petri net if the vector p is positive.
  • check strong coverage: an invariant strongly covers the Petri net if "TODO"
  • export invariants to file: with this option one can set a file to that the calculated invariants are written; this file can be imported by snoopy

T-Invariants

Laxly speaking a T-invariant of a Petri net is a set of transitions that, when they are fired, do not change the marking of the Petri net. Formally, a T-invariant is a vector t so that C t = 0, where C is the incidence matrix.

There are several options for the calculation of invariants.

  • delete trivial invariants: removes all trivial invariants from the list, i.e., only those invariants are listed that are not trivial
  • check coverage: an invariant covers the Petri net if the vector t is positive.
  • check strong coverage: an invariant strongly covers the Petri net if "TODO"
  • export invariants to file: with this option one can set a file to that the calculated invariants are written; this file can be imported by snoopy

Dependent Sets

By enabling the checkbox "dependent sets" once can also calculate the dependent set when an invariant is computed. By default only the abstract depended sets are computed. Further calculations can be set by opening the options menu for dependent sets.

  • strong dependent sets: calculates the strongly dependent sets of the invariant
  • export sds to file: enable this checkbox in order to export the strong dependent sets to a file; a file chooser will be opened
  • abstract dependent sets: calculates the abstract dependent sets of the invariant
  • export ads to file: enable this checkbox in order to export the abstract dependent sets to a file; a file chooser will be opened
  • connected ADS: calculates the connected abstract dependent sets of the invariant
  • export cads to file: enable this checkbox in order to export the connected abstract dependent sets to a file; a file chooser will be opened
